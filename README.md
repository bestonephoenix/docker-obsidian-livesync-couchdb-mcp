# Obsidian LiveSync + MCP Server

**Single Docker container** running **CouchDB for Obsidian LiveSync** and an **MCP SSE server** for AI agent vault access — with end-to-end encryption support.

Agents (Hermes, Claude Desktop, Cursor, etc.) connect to `http://<host>:8000/sse` and get full read/write/search access to your vault. Encryption is unlocked at container startup via a single env var — zero agent involvement.

## How it works

```
┌──────────────────────────────────────────────┐
│                 Docker Container             │
│                                              │
│  ┌──────────┐         ┌───────────────────┐  │
│  │ CouchDB  │◄──────-─│  MCP SSE Server   │  │
│  │  :5984   │ local   │  :8000/sse        │  │
│  │          │  HTTP   │                   │  │
│  └────┬─────┘         └────────┬──────────┘  │
│       │                        │             │
└───────┼────────────────────────┼─────────────┘
        │                        │
   Obsidian clients          AI agents
   (LiveSync plugin)    (Hermes, Claude, etc.)
```

- **CouchDB** (port 5984) — stores your vault, synced by [Obsidian LiveSync](https://github.com/vrtmrz/obsidian-livesync)
- **MCP SSE server** (port 8000) — talks to CouchDB locally, exposes 13 tools over the [Model Context Protocol](https://modelcontextprotocol.io/)
- Both managed by **supervisord** in a single container

## Quick start

```bash
git clone https://github.com/bestonephoenix/docker-obsidian-livesync-couchdb-mcp.git
cd docker-obsidian-livesync-couchdb-mcp

cp .env.example .env
# Edit .env — set COUCHDB_PASSWORD at minimum

docker compose up -d --build
```

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `SERVER_DOMAIN` | No | `localhost` | Domain for CouchDB setup URI |
| `COUCHDB_USER` | No | `admin` | CouchDB admin username |
| `COUCHDB_PASSWORD` | **Yes** | — | CouchDB admin password |
| `COUCHDB_DATABASE` | No | `obsidian` | Database name for the vault |
| `COUCHDB_PORT` | No | `5984` | Host port for CouchDB |
| `MCP_PORT` | No | `8000` | Host port for MCP SSE endpoint |
| `COUCHDB_DATA` | No | `./couchdb-data` | Persistent volume path |
| `LIVESYNC_PASSPHRASE` | No | — | LiveSync E2E encryption passphrase |

## Encryption support

### No encryption? Nothing to configure.

If you don't use LiveSync's end-to-end encryption, the server works out of the box. Chunks without the `e_` flag return as-is.

### Using encryption? Set one env var.

```bash
# In .env:
LIVESYNC_PASSPHRASE=your-passphrase
```

What happens at container startup:

1. Passphrase read from env var
2. PBKDF2 salt **auto-discovered** from `_local/obsidian_livesync_sync_parameters` in CouchDB
3. Credentials stored in memory for the session
4. All encrypted chunks (`e_: true`) decrypted via PBKDF2 → HKDF → AES-256-GCM

The passphrase never leaves the Docker host. No HTTP headers, no query params, no agent tools — the agent never sees or handles it.

### How salt discovery works

The PBKDF2 salt is auto-generated by LiveSync when you first enable encryption and stored base64-encoded in the `pbkdf2salt` field of CouchDB document `_local/obsidian_livesync_sync_parameters`. The server reads it at startup with retry logic (up to 7 attempts, 3s apart) to handle CouchDB startup latency.

If auto-discovery fails (rare), you can set it manually:

```bash
LIVESYNC_PBKDF2_SALT=64-character-hex-string
```

## Connecting agents

### Hermes Agent

```yaml
# ~/.hermes/config.yaml
mcp_servers:
  obsidian:
    url: "http://your-vps:8000/sse"
```

### Claude Desktop

```json
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-client-sse"],
      "env": {
        "MCP_SERVER_URL": "http://your-vps:8000/sse"
      }
    }
  }
}
```

## Available MCP tools

13 tools from [obsidian-self-mcp](https://github.com/suhasvemuri/obsidian-self-mcp):

| Tool | Description |
|---|---|
| `list_notes` | List notes with metadata, filter by folder |
| `read_note` | Read full content of a note |
| `write_note` | Create or update a note |
| `search_notes` | Full-text search across vault |
| `append_note` | Append content to an existing note |
| `delete_note` | Delete a note and its chunks |
| `list_folders` | List folders with note counts |
| `read_frontmatter` | Read YAML frontmatter properties |
| `update_frontmatter` | Set/update frontmatter (JSON input) |
| `list_tags` | List all tags with occurrence counts |
| `search_by_tag` | Find notes containing a tag |
| `get_backlinks` | Find notes linking to a given note |
| `get_outbound_links` | List wikilinks from a note |

## Architecture

### Encryption (when `LIVESYNC_PASSPHRASE` is set)

```
Startup:
  passphrase (env var)
    → discover salt from CouchDB (_local/obsidian_livesync_sync_parameters.pbkdf2salt)
    → _credentials stored in memory

Decryption (per chunk, transparent to agent):
  chunk.data (%= -prefixed base64)
    → PBKDF2(passphrase, pbkdf2_salt, 310k iter, SHA-256) → master key
    → HKDF(master_key, hkdf_salt, SHA-256, empty info) → chunk key
    → AES-256-GCM(chunk_key, IV) → plaintext
```

Key derivation parameters match LiveSync's `octagonal-wheels/hkdf.ts` exactly.

### Container

- **ARM64** by default. For amd64, add `platform: linux/amd64` to docker-compose.
- CouchDB data persists at `/opt/couchdb/data`.
- MCP server connects to CouchDB internally over `localhost:5984`.
- DNS rebinding protection disabled — safe inside Docker's network namespace.
